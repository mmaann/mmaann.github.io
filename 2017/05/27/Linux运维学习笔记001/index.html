centos与Linux的关系，centos与rhel的关系是什么？

centos是Linux的一个发行版本，由centos社区维护。centos是基于rhel的开放源代码二次编译的，出图标外基本与rhel完全一样

各种开源协定的具体细节

  • GPL：软件自始至终都保持开放源代码的形式发布；在软件开放完整源码的情况下，使用者可以自由发挥；软件无论以何种方式发布，都必须附上源代码；开发者可以收取一定服务费，但不能与开放源码捆绑。
  • LGPL：宽松的GPL协定，LGPL是GPL的一个为主要为类库使用设计的开源协议。和GPL要求任何使用/修改/衍生之GPL类库的的软件必须采用GPL协议不同。LGPL允许商业软件通过类库引用(link)方式使用LGPL类库而不需要开源商业软件的代码。这使得采用LGPL协议的开源代码可以被商业软件作为类库引用并发布和销售。
  • BSD：BSD开源协议是一个给于使用者很大自由的协议。基本上使用者可以”为所欲为”，可以自由的使用，修改源代码，也可以将修改后的代码作为开源或者专有软件再发布。
    再次发布的软件或源代码必须包含BSD协定，不能用源代码作者做商业推广

什么是自由软件？与开源软件的关系

根据自由软件基金会的定义，不受限制的自由适用、研究、修改、发布的软件。
自由软件都是开源的，自由软件比开源软件有更严格的概念要求，自由软件的原意是授予人们“自由”适用软件的权利。 开源软件之遵循相关开源协定的软件，不是所有的开源软件都是自由软件。

Linux的哲学思想是什么

  1. 一切皆文件：所有的资源都以文件的形式存在，包括硬件设备
  2. 由众多的小程序组成，每个小程序实现单一功能，复杂功能有多个小程序组合实现
  3. 尽量避免与用户交互，实现脚本化编程，自动实现某些功能
  4. 配置文件以纯文本的方式保存，一款文本编辑器即可实现系统配置
